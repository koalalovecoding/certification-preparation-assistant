# DP-300: Administering Microsoft Azure SQL Solutions (Synthetic)

## Overview

| Field | Details |
|-------|---------|
| Certification Code | DP-300 |
| Certification Name | Administering Microsoft Azure SQL Solutions |
| Type | Single Exam Certification |
| Target Roles | Azure Database Administrator, Data Engineer, Data Scientist |
| Certification Level | Associate |
| Validity Period | 1 year (renewable via free online assessment on Microsoft Learn) |
| Retirement Status | Active — No scheduled retirement date |
| Pass Score | 700 / 1000 |
| Recommended Study Hours | 25 hours |
| Renewal Required | Yes — Annual |
| Renewal Method | Free online assessment on Microsoft Learn |
| Renewal Deadline | None |

## Prerequisites

**Exam Prerequisites:** None — candidates can sit this exam without any prior certification.

**Certification Prerequisites:** None — passing DP-300 directly earns the Microsoft Certified: Azure Database Administrator Associate certification.

---

## Recommended Background

### Responsibilities
Candidates are expected to:
- Manage cloud and on-premises databases built with Azure SQL services and SQL Server
- Implement and manage operational aspects of cloud-native and hybrid data platform solutions
- Perform and automate day-to-day operations using T-SQL and other tools
- Manage database availability, security, performance monitoring and optimization
- Evaluate and implement migration strategies for moving databases between Azure and on-premises

### Proficient In
- Azure SQL Database
- Azure SQL Managed Instance
- SQL Server on Azure Virtual Machines
- SQL Server on-premises
- Transact-SQL (T-SQL)

### Works With
- Azure data engineers
- Azure solution architects
- Azure developers
- Data scientists

### Experience Requirements
- Subject matter expertise in building database solutions with Azure SQL services
- Experience with SQL Server on-premises
- Knowledge of T-SQL for administrative management

## Skills Measured

### 1. Plan and Implement Data Platform Resources (15–20%)

#### Plan and Deploy Azure SQL Solutions
- Recommend a database offering based on specific requirements
- Choose an automated deployment method
- Identify use cases for Azure Arc-enabled SQL services
- Identify use cases for Azure SQL Database in Microsoft Fabric
- Plan for table partitioning and recommend a database sharding solution
- Deploy database offerings on selected platforms
- Deploy hybrid SQL Server solutions
- Apply patches and updates for hybrid and IaaS deployment

#### Configure Resources for Scale and Performance
- Configure Azure SQL Database, Azure SQL Managed Instance, and SQL Server on Azure VMs for scale and performance
- Configure table partitioning and data compression

#### Plan and Implement a Migration Strategy
- Evaluate requirements and strategies for migration
- Implement online and offline migration strategies
- Implement a migration to Azure and between Azure SQL services
- Troubleshoot a migration

---

### 2. Implement a Secure Environment (20–25%)

#### Configure Database Authentication and Authorization
- Configure Microsoft Entra ID authentication for Azure SQL Database and SQL Managed Instance
- Configure security principals and create users from Microsoft Entra identities
- Configure database and object-level permissions
- Apply the principle of least privilege
- Troubleshoot authentication and authorization issues

#### Implement Security for Data at Rest and Data in Transit
- Implement transparent data encryption (TDE)
- Implement object-level encryption
- Configure server- and database-level firewall rules
- Implement Always Encrypted and Always Encrypted with VBS enclaves
- Configure private links and service endpoints

#### Implement Compliance Controls for Sensitive Data
- Apply a data classification strategy
- Configure server and database audits
- Implement change data tracking
- Implement dynamic data masking
- Implement ledger in Azure SQL
- Implement row-level security

---

### 3. Monitor, Configure, and Optimize Database Resources (20–25%)

#### Monitor Resource Activity and Performance
- Prepare an operational performance baseline
- Determine sources for and interpret performance metrics
- Configure and monitor activity and performance
- Monitor by using database watcher and Extended Events

#### Monitor and Optimize Query Performance
- Configure and monitor by using Query Store
- Identify and resolve session blocking
- Identify performance issues using dynamic management views (DMVs)
- Identify and implement index changes for queries
- Review execution plans
- Monitor by using Intelligent Insights

#### Configure Database Solutions for Optimal Performance
- Implement index and statistics maintenance tasks
- Implement database integrity checks
- Configure database automatic tuning
- Configure server settings and Resource Governor for performance
- Configure compute and storage resources for scaling
- Identify use cases for intelligent query processing (IQP) features

---

### 4. Configure and Manage Automation of Tasks (15–20%)

#### Create and Manage SQL Server Agent Jobs
- Manage schedules for regular maintenance jobs
- Configure job alerts and notifications
- Troubleshoot SQL Server Agent jobs

#### Automate Deployment of Database Resources
- Automate deployment by using ARM and Bicep templates
- Automate deployment by using Azure PowerShell and Azure CLI
- Monitor and troubleshoot deployments

#### Create and Manage Database Tasks in Azure
- Create and configure elastic jobs
- Create and configure database tasks by using automation
- Configure alerts and notifications on database tasks
- Troubleshoot automated database tasks

---

### 5. Plan and Configure a High Availability and Disaster Recovery (HA/DR) Environment (20–25%)

#### Plan an HA/DR Strategy
- Recommend HA/DR strategy based on RPO/RTO requirements
- Evaluate HA/DR for hybrid deployments and Azure-specific solutions
- Plan a testing procedure for an HA/DR solution

#### Plan and Perform Backup and Restore
- Recommend a database backup and restore strategy
- Perform database backup and restore by using native tools and T-SQL
- Perform a database restore to a point in time
- Configure long-term backup retention
- Backup to and restore from cloud storage

#### Configure HA/DR for Database Solutions
- Configure active geo-replication
- Configure Always On availability groups on SQL Managed Instance and Azure VMs
- Configure failover groups
- Configure Always On Failover Cluster Instances on Azure VMs
- Configure log shipping
- Monitor and troubleshoot an HA/DR solution
